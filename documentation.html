Open Manual Project

Documentation

An OpenManual document is a simple XML document. There is a schema definition available to validate against and documentation for each tag below.

  • Example XML

Elements of an OpenManual XML Document

manual required

The root node of the OpenManual Document.

Parent Tag: none
Child Tags: title, summary, image, author, time_required, difficulty, categories, introduction, tools, parts, flags, documents, prerequisites, steps, conclusion
  • Attributes:
  • id - optional An integer containing the id of the procedure.
  • locale - required The language the procedure is written in.
  • xmlns - required The site you are getting the source data from. ex: http://www.ifixit.com
  • xmlns:xsi - required Set to http://www.w3.org/2001/XMLSchema-instance
  • xsi:schemaLocation - required The location of the OpenManual schema definition, ex: http://www.ifixit.com/Misc/OpenManual/openmanual_0.2.xsd

title required

Contains the title of the procedure.

Parent Tag: manual
Child Tags: none
  • Attributes:
  • subject - optional The subject of the procedure. ex: Bottom Bezel Bracket
  • type - optional What the procedure explains in relation to the subject. ex: Installation

summary optional

A text field that contains less than 255 characters describing the procedure.

Parent Tag: manual
Child Tags: none
  • Attributes: None

image required

The URL to the image on the front of the procedure.

Parent Tag: manual
Child Tags: none
  • Attributes:
  • guid - optional a global string identifier for the image.
  • author_id - optional an integer identifier for the author of the image.

time_required optional

A description of how long the procedure takes.

ex: 15 minutes

Parent Tag: manual
Child Tags: none
  • Attributes: None

difficulty optional

An arbitrary guess at the difficulty of the procedure.

ex: Difficult.

Parent Tag: manual
Child Tags: none
  • Attributes: None

categories required

A root container for the category tag.

Parent Tag: manual
Child Tags: category
  • Attributes: None

category optional

A single category the procedure belongs to. To specify multiple categories put multiple category tags in the categories tag.

Parent Tag: categories
Child Tags: none
  • Attributes: None

introduction required

The text introducing the procedure. Can be anything from how to set up for the procedure to an introduction of the procedures subject matter.

Parent Tag: manual
Child Tags: none
  • Attributes: None

tools required

A container for the tool tag.

Parent Tag: manual
Child Tags: none
  • Attributes: None

tool

optional - A required or suggested tool.

  • thumbnail - optional an image to go along with the tool.
  • url - optional A URL for a page describing the tool.

parts required

A container for the part tag.

Parent Tag: manual
Child Tags: none
  • Attributes: None

part

optional - A required or suggested part.

  • thumbnail - optional an image to go along with the part.
  • url - optional A URL for a page describing the part.

flags required

A container for the flag tag.

Parent Tag: manual
Child Tags: none
  • Attributes: None

flag

optional - A flag or informational note about the procedure. The text field contains a full text notice of what this flag means to the user.

  • id - optional A unique identifier for the flag.
  • title - required A short title for the flag. ex: In Progress

documents

required - A container for the document tag.

document

optional - A document attached to the procedure. The text for this element should be the filename of the attached document.

  • url - required A URL to the attached document.

prerequisites

required - A container for the prerequisite tag.

prerequisite

optional - The title of a prerequisite procedure. Usually it is assumed the steps of the prerequisites are included in the steps tag.

  • id - required An id for the associated prerequisite procedure.
  • locale - required The language the procedure is written in.

steps

required - A container for the step tag.

step

optional - The step. Contains the lines and images elements.

  • number - required The step number starting from 0.
  • title - optional The title of the step.

lines

required - A container for the line tag.

line

optional - A bullet line of text in the step.

  • bullet - optional Which color and/or type of bullet to use.
  • level - optional How indented this line is.

images

required - A container for the image tag.

image

The URL for an image to include with this step.

  • guid - required a global string identifier for the image
  • orderby - optional The image order.

conclusion

optional - The text for the conclusion.

Open Manual Project is an open source standard being developed by a small group of dedicated individuals.
