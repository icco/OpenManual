Elements of an OpenManual XML Document

guide

required - The root node of the OpenManual Document.

  • id - optional An integer containing the id of the guide.
  • locale - required The language the guide is written in.
  • xmlns - required The site you are getting the source data from. ex. http://www.ifixit.com
  • xmlns:xsi - required Set to http://www.w3.org/2001/XMLSchema-instance
  • xsi:schemaLocation - required The location of the OpenManual schema definition, ex. http://www.ifixit.com/Misc/OpenManual/openmanual_0.2.xsd

title

required - Contains the title of the guide.

  • subject - optional The subject of the guide. ex. Bottom Bezel Bracket
  • type - optional What the guide explains in relation to the subject. ex. Installation

summary

optional - A text field that contains less than 255 characters describing the guide.

image

required - The URL to the image on the front of the guide.

  • guid - optional a global string identifier for the image.
  • author_id - optional an integer identifier for the author of the image.

time_required

optional - A description of how long the guide takes.

ex. 15 minutes

difficulty

optional - An arbitrary guess at the difficulty of the guide.

ex. Difficult.

categories

required - A root container for the category tag.

category

optional - A single category the guide belongs to. To specify multiple categories put multiple category tags in the categories tag.

introduction

required - The text introducing the guide. Can be anything from how to set up for the guide to an introduction of the guides subject matter.

tools

required - A container for the tool tag.

tool

optional - A required or suggested tool.

  • thumbnail - optional an image to go along with the tool.
  • url - optional A URL for a page describing the tool.

parts

required - A container for the part tag.

part

optional - A required or suggested part.

  • thumbnail - optional an image to go along with the part.
  • url - optional A URL for a page describing the part.

flags

required - A container for the flag tag.

flag

optional - A flag or informational note about the guide. The text field contains a full text notice of what this flag means to the user.

  • id - optional A unique identifier for the flag.
  • title - required A short title for the flag. ex. In Progress

documents

required - A container for the document tag.

document

optional - A document attached to the guide. The text for this element should be the filename of the attached document.

  • Url - required A URL to the attached document.

prerequisites

required - A container for the prerequisite tag.

prerequisite

optional - The title of a prerequisite guide. Usually it is assumed the steps of the prerequisites are included in the steps tag.

  • id - required An id for the associated prerequisite guide.
  • locale - required The language the guide is written in.

steps

required - A container for the step tag.

step

optional - The step. Contains the lines and images elements.

  • number - required The step number starting from 0.
  • title - optional The title of the step.

lines

required - A container for the line tag.

line

bullet="icon_caution" level="0"

  • bullet - required Which bullet to use.
  • level - optional How indented this line is.

images

required - A container for the image tag.

image

The URL for an image to include with this step.

  • guid - required a global string identifier for the image
  • orderby - optional The image order.

conclusion

optional - The text for the conclusion.

Open Manual Project is an open source standard being developed by a small group of dedicated individuals.
